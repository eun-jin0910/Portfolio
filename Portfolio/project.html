Oh!GYM

🔎개요

- 헬스 PT(퍼스널트레이닝) 매칭 플랫폼
: 헬스장이나 운동 센터를 방문하지 않고도 온라인으로 트레이너와 상담하며 가격 정보를 제공받아, 편리하게 선택할 수 있도록 도와주는 웹서비스

📌목적

- 현대 사회에서 건강한 삶에 대한 관심과 헬스 관련 산업의 성장, 그리고 바쁜 일상으로 인해 헬스장이나 운동 센터에 가는 것이 부담스러운 현대인들을 대상으로한 편리하고 접근성 높은 서비스를 제공할 수 있는 웹 플랫폼을 개발하고자 함

💻개발환경

- 프로젝트 도구 : Eclipse, VSCode, Github
- 사용 언어 : Java, SQL, Html, Css, JavaScript

📝학습목표

- 기본적인 웹페이지 구성
- 비동기 api 호출, Ajax 비동기 통신에 대한 이해
- jsp/servlet을 활용한 서버 사이드 동적 웹페이지 구현
- Spring MVC에 대한 이해
- 브라우저 개발자 도구 활용

📚담당업무

- 서비스에서 제공할 기능 구체화 및 로직 설계
- 사용할 데이터 파악 및 데이터베이스의 테이블 구조 설계
- 사용자 요구사항 고려하여 전체적인 UI 구상
- 헤더/푸터 UI 구현 및 파일 분리하여 include
- 필요한 서비스를 조회하는 기능 구현
- 근선생 프로필 페이지 UI 구현

💥이슈

1. 페이지 헤더와 푸터에 변경되는 사항이 있을 때, 해당 영역을 동일하게 포함하는 모든 파일을 수정해야 하는 문제 발생
> 해결 : 헤더와 푸터를 컴포넌트화 하여, 각각의 파일에 재사용 및 동적 렌더링
> 결과 : 가독성 증가 및 유지 보수에 용이

2. View들끼리 직접적으로 연결되어 있어 View와 Controller 사이에 역할과 책임이 명확하게 분리되지 않음
> 해결 : View들끼리의 연결이 서블릿을 통해 이루어지게 로직 변경 및 웹 서블릿 매핑을 통해 URL도 변경
> 결과 : 코드의 복잡성이 줄어들고, 유지보수와 확장성 향상. URL의 일관성과 가독성이 높아짐

3. CSR(Client-Side Rendering)로 모든 페이지를 구현하였으나 불필요한 로직과 중복되는 코드로 인해 가독성이 떨어지며, 초기 로딩 속도가 느린 문제 발생
> 해결 : 서버에서 뷰를 렌더링하여 제공하는 SSR(Server-Side Rendering)로 로직 변경
> 결과 : 클라이언트 측에서 처리해야 하는 로직이 줄어들어 가독성 증가 및 초기 로딩 속도 문제 해결
